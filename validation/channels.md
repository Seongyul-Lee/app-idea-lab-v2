# 검증 채널 레퍼런스

Stage 1V에서 사용하는 5가지 검증 방법의 상세 실행 가이드.

---

## A. 랜딩 페이지 테스트

**비용**: 0~5만원 | **기간**: 3~5일 | **난이도**: 중

### 개요

앱이 실제로 존재하는 것처럼 랜딩 페이지를 만들어, "다운로드" 또는 "알림 받기" 버튼 클릭률로 수요를 측정한다.

### 실행 방법

1. **랜딩 페이지 제작** (1일)
   - 도구: Carrd.co (무료), Notion 사이트, 또는 간단한 HTML
   - 필수 요소:
     - 헤드라인: 핵심 가치 한 문장
     - 서브 헤드라인: 구체적 사용 시나리오
     - 앱 목업 이미지 3장 (Figma 또는 AI 생성)
     - CTA 버튼: "출시 알림 받기" (이메일 수집)
     - 간단한 기능 소개 3~4개
   - 절대 금지: "다운로드" 버튼 (아직 앱이 없으므로)

2. **트래픽 유입** (3~5일)
   - 유료: 네이버 검색 광고 (키워드당 일 예산 5,000~10,000원)
   - 무료: 관련 커뮤니티에 "이런 앱 있으면 쓸 건가요?" 형태로 링크 공유
   - SNS: 개인 계정에서 공유

3. **측정 지표**
   - 방문자 수 vs CTA 클릭률 (전환율)
   - 이메일 수집 수

### 성공 기준

| 지표 | 강한 신호 | 보통 | 약함 |
|------|----------|------|------|
| 랜딩 전환율 (방문→이메일) | 10%+ | 5~10% | 5% 미만 |
| 이메일 수집 수 (5일) | 50명+ | 20~50명 | 20명 미만 |

### 팁

- 이메일 수집에는 Tally.so (무료) 또는 Google Forms 사용
- 수집된 이메일은 앱 출시 시 초기 유저 풀이 됨 → Stage 6 Phase 1에서 활용
- 랜딩 페이지는 출시 후 앱스토어 링크로 리디렉트 전환

## B. 커뮤니티 반응 테스트

**비용**: 0원 | **기간**: 1~3일 | **난이도**: 하

### 개요

타겟 유저가 모여있는 커뮤니티에 아이디어를 공유하고 반응을 관찰한다.

### 실행 방법

1. **포스팅 작성** — 3가지 포맷 중 택 1:

   **포맷 1: 문제 공감형**
   ```
   [제목] 저만 이런 건가요? — (pain point 설명)
   [본문] (pain point를 구체적으로 설명)
          혹시 이런 경험 있으신 분? 어떻게 해결하고 계세요?
          (절대 앱 이야기 안 함)
   ```
   → 댓글에서 공감/솔루션 니즈 확인

   **포맷 2: 솔루션 제안형**
   ```
   [제목] (pain point) 해결하는 앱 만들어보려는데 의견 부탁드립니다
   [본문] (아이디어 간략 설명)
          이런 앱 있으면 쓰실 건가요?
          어떤 기능이 가장 중요할까요?
   ```

   **포맷 3: 투표형**
   ```
   [제목] (pain point) 관련 — 어떤 기능이 가장 필요한가요?
   [투표] 기능 A / 기능 B / 기능 C / 필요 없음
   ```

2. **게시판 선정** — 타겟에 맞는 3곳 이상
3. **반응 수집** — 48시간 동안 모니터링

### 성공 기준

| 지표 | 강한 신호 | 보통 | 약함 |
|------|----------|------|------|
| 댓글/반응 수 | 20개+ | 5~20개 | 5개 미만 |
| 긍정 비율 | 70%+ | 40~70% | 40% 미만 |
| "출시되면 알려달라" 반응 | 5명+ | 1~5명 | 0명 |

### 주의사항

- 커뮤니티 규칙 반드시 확인 (홍보 금지 게시판 다수)
- 광고/홍보처럼 보이면 역효과 → "순수한 질문/의견 수렴" 톤 유지
- 부정적 반응도 귀중한 데이터. 방어하지 말고 기록할 것

## C. 가짜 문 테스트 (Fake Door Test)

**비용**: 0원 | **기간**: 1~2일 | **난이도**: 하

### 개요

기존에 운영 중인 앱이 있다면, 해당 앱 내에 "새 기능 출시 예정" 배너를 넣어 클릭률을 측정한다. 기존 앱이 없으면 B 방법 사용.

### 실행 방법

1. 기존 앱 내 적절한 위치에 배너/버튼 추가:
   ```
   "🔔 [새 기능명] 출시 예정! 미리 알림 받기"
   ```
2. 클릭 시 → "준비 중입니다. 알림을 원하시면 이메일을 남겨주세요"
3. 클릭률 + 이메일 수집 수 측정

### 성공 기준

| 지표 | 강한 신호 | 약함 |
|------|----------|------|
| 클릭률 (노출 대비) | 5%+ | 2% 미만 |

## D. 설문조사

**비용**: 0~3만원 | **기간**: 3~5일 | **난이도**: 중

### 개요

타겟 유저 30~100명에게 설문을 돌려 pain point 강도와 지불 의사를 확인한다.

### 실행 방법

1. **설문 도구**: Google Forms, Tally.so (무료)
2. **필수 질문** (10문항 이내):

   ```
   Q1. [pain point]를 얼마나 자주 경험하시나요?
       매일 / 주 2~3회 / 월 2~3회 / 거의 안 함

   Q2. 현재 이 문제를 어떻게 해결하고 계신가요?
       (주관식)

   Q3. 현재 해결 방법에 얼마나 만족하시나요?
       매우 만족 / 만족 / 보통 / 불만족 / 매우 불만족

   Q4. [앱 컨셉 한 줄 설명]
       이런 앱이 있다면 사용하시겠습니까?
       반드시 사용 / 아마 사용 / 모르겠음 / 아마 사용 안 함 / 절대 사용 안 함

   Q5. 이 앱에 월 얼마까지 지불하실 의향이 있으신가요?
       무료만 사용 / ~1,000원 / ~3,000원 / ~5,000원 / ~10,000원 / 10,000원+

   Q6. 가장 중요한 기능 1가지를 고르신다면?
       기능 A / 기능 B / 기능 C

   Q7. 나이대
   Q8. 성별
   Q9. 직업군
   Q10. 기타 의견 (주관식)
   ```

3. **배포 채널**: 커뮤니티, SNS, 지인 네트워크
4. **인센티브** (선택): 스타벅스 기프티콘 추첨 (3만원 예산)

### 성공 기준

| 지표 | 강한 신호 | 보통 | 약함 |
|------|----------|------|------|
| Q4 "반드시 사용" 비율 | 30%+ | 15~30% | 15% 미만 |
| Q5 유료 의향 (1,000원+) | 40%+ | 20~40% | 20% 미만 |
| 응답 수 | 50명+ | 30~50명 | 30명 미만 |

### 주의사항

- "사용하시겠습니까?"에 "네"라고 답한 사람 중 실제 다운로드는 10~20%
- 유료 의향도 실제 결제는 1/3~1/5 수준으로 보수적으로 해석할 것
- 주관식 답변이 정량 데이터보다 더 귀중한 경우가 많음

## E. 검색 광고 테스트

**비용**: 5~20만원 | **기간**: 3~5일 | **난이도**: 상

### 개요

네이버/구글 검색 광고를 소액 집행하여, 해당 키워드의 실제 클릭률과 전환 의향을 측정한다.

### 실행 방법

1. **랜딩 페이지 준비** (방법 A와 동일)
2. **네이버 검색 광고 세팅**:
   - 키워드 5~10개 등록
   - 일 예산: 10,000~20,000원
   - 기간: 3~5일
3. **구글 검색 광고 세팅** (선택):
   - 키워드 5~10개
   - 일 예산: 10,000원
   - 기간: 3~5일
4. **측정**: 클릭률(CTR), 랜딩 페이지 전환율, CPC(클릭당 비용)

### 성공 기준

| 지표 | 강한 신호 | 보통 | 약함 |
|------|----------|------|------|
| CTR (검색 광고) | 3%+ | 1~3% | 1% 미만 |
| 랜딩 전환율 | 10%+ | 5~10% | 5% 미만 |
| CPC | 500원 이하 | 500~1,000원 | 1,000원+ |

### 이 방법의 장점

- 가장 정확한 수요 측정 방법 (실제 돈을 써서 검색하는 사람이 대상)
- CPC가 낮으면 = 경쟁이 적고 수요가 있다 = 좋은 기회
- 수집된 키워드 데이터는 Stage 5 ASO에서 그대로 활용 가능

---

## 방법 선택 가이드

```
아이디어 특성에 따라:

니치 커뮤니티가 명확함? ─── Yes → B (커뮤니티) + D (설문)
                          └── No ─┐
                                   │
검색 키워드가 명확함? ──── Yes → E (검색 광고) + A (랜딩)
                          └── No ─┐
                                   │
기존 앱이 있음? ────────── Yes → C (가짜 문)
                          └── No → A (랜딩) + B (커뮤니티)

예산 0원 제약? ─────────── Yes → B (커뮤니티) + D (설문, 인센티브 없이)
```
