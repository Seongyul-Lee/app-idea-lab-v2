# 한국 식단/영양 관리 앱 시장 현황 조사

> 조사일: 2026-01-30

---

## 1. 한국 주요 식단/영양 관리 앱 현황

### 1.1 밀리그램 (Milligram)
- **운영사**: 주식회사 킬로
- **출시**: 2020년
- **사용자**: 2030 여성 중심, 누적 170만+ 다운로드, 8,000만 건 기록 데이터
- **평점**: 4.8점 (1만 6천 건 리뷰)
- **수상**: Google Play 2022년 '올해를 빛낸 앱'
- **핵심 기능**:
  - AI 코칭 기능 (2024년 출시): 식단 사진 -> AI가 칼로리/탄단지 자동 분석
  - 식단/운동/신체/물/수면/컨디션/생리 기록
  - 그룹 다이어트 (실시간 기록 공유, 멤버 응원)
  - 캘린더 자동 연동, 통계 그래프
- **수익 모델**: AI 코칭 구독, 프리미엄 기능
- **사용자 불만**:
  - 그룹 기능 추가 시 기존 '모아보기' UX 위치 변경으로 사용성 저하
  - 사진 미등록 시 UI 표시 불편
  - 혜택(포인트) 기능 종료

### 1.2 다이어트신
- **특징**: 한국 음식 칼로리 DB 최대 보유
- **사용자**: 꾸준한 사용자층
- **핵심 기능**:
  - 국내 최대 한국 음식 칼로리 데이터베이스 (김치찌개~편의점 도시락)
  - 국내 최대 다이어트 커뮤니티 운영
  - 의사/영양사 전문가의 다이어트 팁 제공
  - 다이어리, 칼로리 사전, 만보기, 물알림
- **수익 모델**: 100% 무료 (광고 기반)
- **강점**: 한국 음식 특화, 완전 무료
- **약점**: AI 기반 자동 분석 기능 부재, 수기 입력 의존

### 1.3 팻시크릿 (FatSecret)
- **특징**: 글로벌 무료 칼로리 카운터
- **핵심 기능**:
  - 바코드 스캐너 + 매뉴얼 바코드 입력
  - Samsung Health, Google Fit, Fitbit 연동
  - 운동 다이어리
- **수익 모델**: 핵심 기능 무료
- **사용자 불만**: 기능 사용이 직관적이지 못하고 어려움, 음식 등록 부족

### 1.4 삼성 헬스 (Samsung Health)
- **사용자**: MAU 약 1,021만 명 (2025년 7월 기준, 국내)
- **식단 기능**:
  - FatSecret + 한국영양학회 DB 사용
  - 바코드 스캔 기능 추가
  - 영양소 비율 분석
- **주요 불만 사항**:
  - 한국 음식/외식 데이터 부족 (등록되지 않은 음식 많음)
  - 비타민/무기질 등 영양 정보 부정확 또는 누락 (비타민 A/C가 0%로 표시되는 등)
  - 체중관리 통합 기능 삭제 (섭취 칼로리 vs 소모 칼로리 비교 화면)
  - 탄단지 g 표시 제거, 비율만 표시로 변경
  - 미래 식단 계획 기능 없음 (이미 먹은 것만 기록 가능)
  - 미량 영양소 기록 제한

### 1.5 눔 (Noom)
- **특징**: 심리학 기반 코칭 앱 (한국 출신 글로벌 기업)
- **사용자**: 전 세계 4,500만+ 회원, 여성 비율 67.5%
- **핵심 기능**:
  - 건강관리 코치의 모니터링 + 메시지 조언
  - 식단/운동/체중/혈당/혈압 기록
  - 심리학 기반 행동 변화 프로그램
- **성과**: 65% 사용자가 1년 후에도 감량 체중 유지
- **수익 모델**: 유료 구독 (고가)

### 1.6 카카오 파스타 (PASTA)
- **운영사**: 카카오헬스케어
- **특징**: AI 기반 혈당/체중 관리 서비스
- **핵심 기능**:
  - 연속혈당측정기(CGM) 연동 실시간 혈당 관리
  - '푸드샷' AI 음식 인식: 검출기 + 분류기 + 판별기 3단계 구조
  - 주변 음식 컨텍스트 활용 (옆에 단무지 -> 짬뽕 인식)
  - 20가지 디지털 피노타입 기반 초개인화 관리
- **주요 한계**:
  - 인식 가능한 음식 종류 한정적
  - 섭취량 기반 칼로리 자동 계산 미흡
  - 애플 피트니스 앱 연동 불가 (개선 중)
  - 시장에서 이용자 수 확보에 어려움

### 1.7 기타 주목할 앱
- **인아웃**: 50만+ 음식 영양 정보, AI 칼로리 계산, 다이어트 커뮤니티
- **Cal AI**: 사진 촬영 -> AI 자동 분석, 정확도 80~90%, 17세 창업자가 월 14억 매출
- **YAZIO**: AI 칼로리 계산기, 무료 사용 시 핵심 액션 중 광고 노출 불만
- **MyFitnessPal**: 1,400만+ 음식 DB 글로벌 1위, 한국 음식 데이터 부족
- **지니어트**: 홈트+다이어트+혈당기록+만보기, 114만 사용자

## 2. 한국 음식 DB 기반 맞춤형 서비스의 현황과 한계

### 2.1 공공 데이터 현황

#### 식품의약품안전처 식품영양성분 DB
- 관계기관 협업 통합 DB (식약처, 농촌진흥청, 해양수산부)
- 식품별 24개 영양성분 정보 제공
- 분기별 업데이트
- 약 17,177종 식품 항목 함량 정보 포함
- Open API 제공 (공공데이터포털)

#### 전국통합식품영양성분정보(음식) 표준데이터
- 국민건강영양조사 음식별 식품재료량 자료집 기반
- 식품코드, 식품명, 에너지(kcal), 수분, 단백질, 지방, 탄수화물, 당류, 식이섬유, 칼슘, 철, 나트륨, 비타민 등
- 전체 데이터 API 활용 가능

#### 식품영양성분 DB 통합 자료집
- 548 품목의 영양정보 (실제 음식 다양성 대비 부족)
- 기준 레시피 4개 기관(한국문화재단, 농촌진흥청, 식약처, 한식재단) 레시피 활용
- 3회 분석 평균값으로 영양성분 산출, 관능평가 검증

#### 국가표준식품성분표 (농촌진흥청)
- 한국인 다소비 식품의 영양성분, 기능성분, 전통음식 정보
- 무료 식단 작성 기능 제공

### 2.2 주요 한계

| 한계 | 상세 |
|------|------|
| **기관별 DB 분산** | 식약처, 농진청, 질병관리청, 한국영양학회 등이 각각 별도 DB 운영. 목적/영양소 항목 상이 |
| **레시피 표준화 어려움** | 같은 음식도 가정/지역/업소별 조리법이 달라 표준 레시피 설정이 근본적 난제 |
| **분석 품목 수 제한** | 통합 자료집 548품목. 실제 국민이 섭취하는 음식 다양성 대비 크게 부족 |
| **조리법 복잡성** | 한식의 습열조리, 양념 사용, 한약재 첨가 등 복합적 조리 특성 |
| **외식/가공식품 미반영** | 프랜차이즈, 편의점 식품 등의 정확한 영양 정보 부족 |
| **영양소 정보 불완전** | 비타민, 무기질 등 미량 영양소 데이터 누락 다수 |

## 3. 사용자 불만 종합

### 3.1 수기 입력의 피로감 (가장 큰 불만)
- 매끼 음식을 검색해서 선택/입력하는 과정이 번거로움
- 등록되지 않은 음식은 직접 등록해야 함
- 샐러드 하나도 야채/소스/토핑 따로 입력 필요
- 노년층이나 입력에 불편함을 느끼는 사용자 이탈 원인
- AI 사진 인식이 해결책으로 부상하나, 한국 음식 정확도 문제

### 3.2 한국 음식 데이터 정확도
- 같은 음식이 앱마다 칼로리가 다르게 나옴
- 글로벌 앱(MyFitnessPal, YAZIO)은 한국 음식 데이터 부족
- 국내 앱도 외식/가공식품 정보 불완전
- 조리법에 따른 영양소 차이 미반영

### 3.3 AI 사진 인식 정확도
- 한국 음식의 mixed food 특성 (찌개, 비빔밥 등)으로 인식 어려움
- 여러 반찬이 함께 놓이는 상차림에서 음식 간 영역 겹침 문제
- 인식 가능한 음식 종류 한정적
- 섭취량(양) 추정의 어려움

### 3.4 기능/UX 관련 불만
- 칼로리만 제공하고 종합적 영양 평가 부족
- 미래 식단 계획 기능 없음 (삼성 헬스)
- 탄단지 g 표시 제거 (삼성 헬스)
- 체중관리 통합 뷰 삭제 (삼성 헬스)
- 핵심 기록 동작 중 광고 삽입 (YAZIO)

### 3.5 지속성 문제
- 식단 기록의 귀찮음으로 인한 이탈률 높음
- 동기 부여 체계 부족
- 눔만이 심리학 기반 접근으로 65% 장기 유지율 달성

## 4. 2026 '건강지능(HQ)' 트렌드와의 연관성

### 4.1 건강지능(HQ) 개념
- 《트렌드 코리아 2026》(김난도 교수) 선정 2026년 10대 키워드
- "내 몸을 얼마나 잘 알고 있는가" — 수면, 운동, 식단, 스트레스 관리 능력
- 핵심: **예방**과 **개인 맞춤형 관리**
- 질병 발생 전 건강 상태 수시 모니터링 및 예방

### 4.2 식단/영양 관리 앱과의 연관성
- 건강지능의 핵심 실천 영역 중 하나가 **식단 구성 및 영양 관리**
- 스마트워치/헬스 앱으로 내 몸의 데이터를 직접 관리/분석하는 트렌드 부상
- 넘쳐나는 건강 정보에서 **내 몸에 맞는 진짜 정보**를 찾는 것이 핵심
- 단순 칼로리 카운팅 -> 개인 맞춤 영양 최적화로 패러다임 전환

### 4.3 관련 글로벌 웰니스 트렌드 (더퓨처 2026 키워드)
- **Optimized AI Nutrition**: AI 최적화 영양 관리가 2026 핵심 트렌드
- **Lifestyle Re-Coding**: 생활습관 리코딩
- **Exercise Intelligence**: 운동 지능
- **Longevity**: 단순 장수 -> 신체/정신 건강 유지 활력 있는 삶

### 4.4 시장 기회
- 국내 웰니스 산업 규모: 약 1,130억 달러 (2022년, 세계 9위)
- Z세대 83%가 헬스케어 앱 이용 중, 유료 이용 중 35.6%가 다이어트 앱
- 식품/뷰티/디지털 헬스케어 업계에서 건강지능 마케팅 활발
- 혈당관리 다이어트 트렌드 급부상 (2025~2026)

### 4.5 핵심 인사이트
> 건강지능 트렌드의 본질은 "내 몸에 맞는 맞춤형 건강 관리"이며, 현재 식단 앱 시장은 **범용적 칼로리 카운팅**에 머물러 있다. **한국 음식 특화 + 개인 맞춤 영양 분석**이 가능한 앱이 이 트렌드에 부합하는 기회를 포착할 수 있다.

## 5. 한국 음식 특화 영양 관리의 기술적 난이도

### 5.1 음식 사진 인식 (Food Image Recognition)

#### 난이도: 높음

**한국 음식 인식이 어려운 이유**:
1. **Mixed food 특성**: 다양한 양념/조리법이 혼합된 음식 (찌개, 비빔밥, 잡채 등)
2. **복잡한 상차림**: 여러 반찬이 함께 놓이는 한식 상차림 (서양식 메인 1접시와 근본적으로 다름)
3. **음식 간 영역 겹침**: 검출 모델이 음식 박스 영역이 옆 음식과 겹쳐 잘못 분류
4. **외형 유사 음식 다수**: 붉은 국물 음식(짬뽕 vs 육개장 vs 김치찌개 등) 구분 어려움
5. **학습 데이터 부족**: 공개된 한국 음식 이미지 DB가 부족

**현재 기술 수준**:
| 프로젝트/기관 | 인식 종류 수 | 정확도 | 비고 |
|---|---|---|---|
| 카카오엔터프라이즈 AI Lab | 1,000여 종 | 미공개 | 검출기+분류기 구조, 컨텍스트 활용 |
| AI Hub 데이터셋 | 500개 카테고리 | - | 카테고리별 약 5만 장 |
| AI Hub 한국 음식 이미지 | 150종 | - | 종별 약 1천 장, 세그먼트 태깅 |
| 당뇨환자 영양관리 연구 | 2,000개 | 83% | CNN + Fine Tuning + Augmentation |
| 카카오 파스타 푸드샷 | 미공개 | 미공개 | 클래스별 최소 200+ 이미지, 컨텍스트 인식 |
| Cal AI (글로벌) | - | 80~90% | 한국 음식도 비교적 양호 |

**1인 개발 시 현실적 접근**:
- 자체 음식 인식 AI 모델 개발: **사실상 불가** (대규모 데이터 수집/학습 필요)
- 기존 AI API 활용 (Google Vision, OpenAI GPT-4V 등): **가능하나 정확도 제한**
- AI Hub 공개 데이터셋 활용: **가능하나 추가 학습/파인튜닝 필요**
- GPT-4 Vision 등 멀티모달 LLM 활용: **가능, 한국 음식 인식 정확도 검증 필요**

### 5.2 한국 음식 영양 DB 구축/연동

#### 난이도: 중간

**활용 가능한 공공 데이터**:
1. **식약처 식품영양성분 DB**: Open API 제공, 17,177종, 24개 영양성분
2. **전국통합식품영양성분정보 표준데이터**: API 활용 가능
3. **국가표준식품성분표**: 농촌진흥청 제공

**기술적 과제**:
- 기관별 DB 데이터 형식 통합/정제 필요
- 동일 음식의 기관별 영양소 값 차이 처리
- 조리법/레시피 변동에 따른 영양소 보정 로직
- 외식/가공식품/프랜차이즈 메뉴 데이터 추가 수집 필요
- 섭취량(1인분 기준) 표준화

**1인 개발 시 현실적 접근**:
- 공공 API 연동: **가능** (React Native에서 REST API 호출)
- 커뮤니티 기반 음식 DB 확장: **가능** (사용자 기여 데이터)
- 외식/가공식품 바코드 스캔 연동: **가능** (FatSecret API 등 활용)

### 5.3 개인 맞춤 영양 분석

#### 난이도: 중간~높음

**필요 기술**:
- 개인 건강 정보(나이, 성별, 활동량, 건강 목표) 기반 영양소 권장량 계산
- 식단 기록 데이터 기반 영양소 과잉/결핍 분석
- AI 기반 식단 추천 (LLM 활용 가능)
- 혈당 연동 시 CGM 기기 블루투스 연동 (높은 난이도)

**1인 개발 시 현실적 접근**:
- 기본 영양소 분석 (탄단지/칼로리/나트륨 등): **가능**
- LLM 기반 식단 조언: **가능** (OpenAI API 등 활용)
- CGM 연동: **어려움** (기기별 SDK 필요, 인증 문제)

## 8. 핵심 발견 및 시사점

### 시장 갭 (기회 영역)
1. **한국 음식 특화 + AI 자동 인식의 결합 부재**: 다이어트신은 한국 음식 DB는 강하나 AI 인식 없음. AI 인식 앱들은 한국 음식 정확도 부족
2. **개인 맞춤 영양 분석 부재**: 대부분 칼로리 카운팅에 머물러 있고, 개인 건강 상태에 맞는 영양소 분석/추천은 미흡
3. **식단 기록 지속성 문제 미해결**: 수기 입력 피로감으로 이탈률 높음. AI 사진 인식이 해결책이나 한국 음식 정확도 과제
4. **건강지능(HQ) 트렌드와의 정합성**: "내 몸에 맞는 맞춤 관리"를 실현하는 앱은 아직 부재

### 주의 사항
- 음식 사진 인식 자체 개발은 1인 개발로 불가 (외부 API/LLM 활용 필수)
- 카카오, 삼성 등 대기업도 음식 인식 정확도와 이용자 확보에 고전 중
- 혈당 관리(CGM 연동)는 기기 의존성/인증 문제로 1인 개발 부적합
