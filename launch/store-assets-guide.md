# 스토어 에셋 제작 가이드

1인 개발자를 위한 앱스토어/플레이스토어 에셋 제작 실전 가이드. 디자인 전문가가 아니어도 전환율 높은 스토어 페이지를 만드는 방법.

---

## 1. 앱 아이콘 제작

### 원칙

- **1가지 핵심 요소만** — 아이콘에 2개 이상의 요소를 넣지 말 것
- **원거리 테스트** — 1cm 크기에서도 식별 가능해야 함
- **경쟁앱과 색상 차별화** — 카테고리 Top 20 아이콘 색상을 조사한 뒤, 가장 적은 색상 선택
- **그라데이션 배경** — 단색보다 고급스러운 느낌. 미세한 그라데이션 권장
- **글자 사용 최소화** — 1~2글자 이니셜 정도만 (앱 이름을 아이콘에 풀네임으로 쓰지 말 것)

### 제작 도구

| 도구 | 비용 | 용도 |
|------|------|------|
| Figma | 무료 | 직접 디자인 |
| Canva | 무료/유료 | 템플릿 기반 |
| Midjourney / DALL-E | $10~30/월 | AI 생성 후 Figma에서 정리 |
| Recraft.ai | 무료 | 아이콘 특화 AI 생성 |

### 사이즈

| 플랫폼 | 사이즈 | 포맷 |
|--------|--------|------|
| iOS | 1024×1024px | PNG (투명 배경 불가) |
| Android | 512×512px | PNG (32-bit) |

## 2. 스크린샷 제작

### 레이아웃 구조

```
┌─────────────────────────┐
│     캡션 텍스트 (상단 1/3)  │ ← 큰 글씨, 2줄 이내
│     18~24pt, Bold         │
│                           │
├─────────────────────────┤
│                           │
│     기기 목업 + 앱 화면     │ ← 실제 앱 스크린샷
│     (하단 2/3)             │
│                           │
│                           │
│                           │
└─────────────────────────┘
     배경: 앱 브랜드 컬러
```

### 캡션 작성 공식

❌ 기능 나열:
```
"복약 알림 기능"
"가족 공유 기능"
"복약 통계 기능"
```

✅ 혜택 전달:
```
"매일 정해진 시간에 약을 놓치지 마세요"
"가족의 복약 현황을 한눈에"
"한 달 복약 기록을 한 장으로"
```

### 6장 스토리라인 템플릿

| 순서 | 캡션 공식 | 화면 |
|------|----------|------|
| 1 | "[결과/혜택]을 [방법]으로" | 앱의 대표 화면 (가장 예쁜 화면) |
| 2 | "[pain point], 이제 끝" | 핵심 기능 #1 사용 화면 |
| 3 | "[경쟁앱에 없는 것]까지" | 차별화 기능 화면 |
| 4 | "[편의 기능]도 한번에" | 부가 기능 화면 |
| 5 | "어디서든 [핵심 가치]" | 위젯/알림/워치 화면 |
| 6 | "지금 시작하세요" 또는 별점/리뷰 수 | 온보딩 또는 사회적 증거 |

### 제작 도구

| 도구 | 비용 | 특징 |
|------|------|------|
| **Previewed.app** | 무료/유료 | 기기 목업 + 스크린샷 합성 자동화. 가장 추천 |
| **Rotato** | 유료 ($49) | 3D 기기 목업. 고퀄리티 |
| **Figma** | 무료 | 직접 레이아웃. 자유도 높음 |
| **AppMockUp** | 무료 | 웹 기반 목업 생성기 |
| **Canva** | 무료 | 템플릿 다수 |

### 사이즈 규격

**iOS (필수)**:
| 기기 | 사이즈 | 비고 |
|------|--------|------|
| 6.7인치 (iPhone 15 Pro Max) | 1290 × 2796 | 필수 |
| 5.5인치 (iPhone 8 Plus) | 1242 × 2208 | 필수 |
| iPad Pro 12.9인치 | 2048 × 2732 | iPad 지원 시 |

**Android**:
| 최소 사이즈 | 최대 사이즈 |
|------------|-----------|
| 320 × 320 | 3840 × 3840 |

→ 권장: 1080 × 1920 (16:9 세로)

## 3. 앱 설명 작성 템플릿

### 한국어 앱 설명 구조

```markdown
[핵심 가치 한 문장] — 유저가 얻는 결과
[구체적 시나리오] — "이런 상황에서 이렇게 쓰세요"
[차별점 또는 사회적 증거]

━━━━━━━━━━━━━━━

▸ [혜택 1]
  (1~2줄 부연 설명)

▸ [혜택 2]
  (1~2줄 부연 설명)

▸ [혜택 3]
  (1~2줄 부연 설명)

▸ [혜택 4]
  (1~2줄 부연 설명)

━━━━━━━━━━━━━━━

💎 프리미엄 구독
(구독 혜택 간략 설명)
· 월간: ₩X,XXX
· 연간: ₩XX,XXX (XX% 할인)

━━━━━━━━━━━━━━━

📮 문의: support@example.com
📋 이용약관: https://...
🔒 개인정보 처리방침: https://...
```

### 작성 규칙

- 첫 3줄 = 스토어에서 "더 보기" 클릭 전 보이는 전부 → 가장 공들일 것
- Primary 키워드 3~5회 자연 포함 (억지 반복 금지)
- "~할 수 있습니다" → "~하세요" 로 변환 (행동 유도형)
- 이모지: 구분자로만 사용 (▸, ✓, 💎 등). 문장 안에 이모지 금지
- 한 문단 3줄 이내. 스마트폰에서 읽기 쉬운 길이

## 4. 프로모션 그래픽 (Play Store)

### 사이즈

1024 × 500px (PNG 또는 JPG)

### 구성

```
┌─────────────────────────────────────────────────┐
│                                                   │
│    [앱 아이콘]  [캐치프레이즈]                      │
│               [한 줄 설명]                         │
│                                                   │
└─────────────────────────────────────────────────┘
```

- 앱 아이콘을 좌측 1/3에 배치
- 캐치프레이즈를 우측 2/3에 배치
- 배경은 앱 브랜드 컬러 (스크린샷과 통일)
- 피처링될 경우 이 이미지가 대형으로 노출됨

---

## 5. 앱 미리보기 영상 (선택)

### 기본 규칙

- 길이: 15~30초 (최대 30초)
- 처음 3초에 핵심 장면 (자동 재생이므로)
- 텍스트 오버레이로 핵심 혜택 전달
- BGM: 저작권 무료 음악 (Pixabay, YouTube Audio Library)
- 해상도: 스크린샷과 동일 (1290×2796 등)

### 제작 방법 (1인 개발자용)

1. 앱 실제 사용 화면을 화면 녹화
2. 기기 목업에 합성 (Rotato 추천)
3. 텍스트 캡션 + 전환 효과 추가 (CapCut 무료)
4. 15~30초로 편집
