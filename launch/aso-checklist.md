# ASO 체크리스트

앱 출시 전/후 반드시 점검해야 할 ASO(App Store Optimization) 항목.
스토어 제출 전에 모든 ✅ 항목을 완료할 것.

---

## 1. 앱 메타데이터

### App Store (iOS)

- [ ] **앱 이름** (30자): Primary 키워드 1개 이상 포함
- [ ] **부제목** (30자): 앱 이름과 중복되지 않는 Secondary 키워드 포함
- [ ] **키워드 필드** (100자): 쉼표로 구분, 앱 이름/부제목과 중복 금지, 복수형 불필요
- [ ] **프로모션 텍스트** (170자): 심사 없이 변경 가능. 이벤트/업데이트 알림용
- [ ] **설명** (4,000자): 첫 3줄에 핵심 가치 + Primary 키워드 자연 포함
- [ ] **새로운 기능** (4,000자): 업데이트 내용 구체적 작성 (리뷰 유도 효과)
- [ ] **카테고리**: Primary + Secondary 설정. 경쟁 적은 카테고리 선택
- [ ] **콘텐츠 등급**: 정확한 등급 설정 (앱 내용에 맞게)
- [ ] **개인정보 처리방침 URL**: 필수
- [ ] **지원 URL**: 필수

### Play Store (Android)

- [ ] **앱 이름** (30자): Primary 키워드 포함
- [ ] **짧은 설명** (80자): 핵심 가치 + 키워드 (iOS 부제목+키워드 필드 역할)
- [ ] **전체 설명** (4,000자): 키워드 자연 포함 (3~5회), 처음 80자가 가장 중요
- [ ] **카테고리**: 적합한 카테고리 1개
- [ ] **콘텐츠 등급**: IARC 설문 완료
- [ ] **개인정보 처리방침 URL**: 필수
- [ ] **데이터 안전 섹션**: 모든 데이터 수집/공유 항목 정확히 기재

## 2. 비주얼 에셋

### 앱 아이콘

- [ ] iOS: 1024×1024px (PNG, 투명 배경 불가)
- [ ] Android: 512×512px (PNG, 32-bit)
- [ ] 단순한 디자인 (작은 사이즈에서도 식별 가능)
- [ ] 글자 최소화 (1~2글자 이내, 없는 것이 이상적)
- [ ] 경쟁앱 아이콘과 차별화되는 색상/형태
- [ ] 배경에 사진 사용 금지

### 스크린샷

- [ ] **iOS 필수 사이즈**: 6.7인치 (1290×2796), 5.5인치 (1242×2208)
- [ ] **Android 필수**: 최소 2장, 권장 8장
- [ ] 최소 6장 (iOS), 8장 (Android) 준비
- [ ] 1장: 히어로 샷 (핵심 가치 한 문장)
- [ ] 2~5장: 핵심 기능별 혜택 설명
- [ ] 마지막 장: CTA 또는 사회적 증거
- [ ] 캡션 텍스트: 10단어 이내, 큰 글씨
- [ ] 기기 목업 사용 (선택이지만 권장)
- [ ] 브랜드 일관성 (색상, 폰트 통일)

### 추가 에셋

- [ ] **Play Store 프로모션 그래픽**: 1024×500px (피처링 대비 필수)
- [ ] **앱 미리보기 영상** (선택): 15~30초, 처음 3초에 핵심 장면

---

## 3. 기술 점검

- [ ] **최소 iOS 버전**: 16.0 이상 권장 (시장 커버리지 95%+)
- [ ] **최소 Android 버전**: API 26 (Android 8.0) 이상
- [ ] **빌드 테스트**: Release 빌드로 실 기기 테스트 완료
- [ ] **크래시 없음**: 주요 플로우 5개 이상 크래시 없이 동작
- [ ] **딥링크/유니버설 링크**: 설정 필요 시 동작 확인
- [ ] **푸시 알림**: 정상 수신 확인 (iOS/Android 모두)
- [ ] **인앱 구매**: Sandbox/테스트 환경에서 결제 플로우 정상 동작
- [ ] **개인정보**: 불필요한 권한 요청 제거
- [ ] **다크 모드**: 지원 시 깨지는 화면 없는지 확인
- [ ] **접근성**: VoiceOver/TalkBack 기본 동작 확인

---

## 4. 수익화 설정

- [ ] **AdMob** (광고 사용 시): 앱 ID, 광고 단위 ID 프로덕션 키로 교체
- [ ] **인앱 구매** (iOS): App Store Connect에서 상품 등록 + 심사 제출
- [ ] **인앱 구매** (Android): Play Console에서 상품 등록
- [ ] **구독 상품**: 무료 체험 기간 설정 (3일 또는 7일 권장)
- [ ] **가격**: 한국 시장 기준으로 설정 완료
- [ ] **결제 복원**: "구매 복원" 버튼 동작 확인 (iOS 필수)

---

## 5. 분석/추적 설정

- [ ] **Analytics**: Aptabase 또는 Firebase Analytics 초기화 확인
- [ ] **핵심 이벤트 정의 + 트래킹**:
  - 앱 설치 후 첫 실행
  - 온보딩 완료
  - 핵심 기능 첫 사용
  - 구매/구독 시작
  - 구독 해지
- [ ] **크래시 리포팅**: Sentry 또는 Crashlytics 설정
- [ ] **리뷰 요청 트리거**: 조건 설정 (3일+ 사용, 핵심 기능 2회+ 사용)

---

## 6. 출시 전 최종 체크

- [ ] 앱 내 하드코딩된 테스트 데이터 제거
- [ ] 개발용 로그/디버그 모드 비활성화
- [ ] .env 파일의 프로덕션 키 확인
- [ ] Supabase RLS 정책 프로덕션 환경 확인
- [ ] 이용약관 + 개인정보 처리방침 페이지 라이브
- [ ] 고객 지원 이메일 설정 (전용 이메일 권장)
- [ ] 앱 내 "문의하기" 동작 확인
- [ ] 스토어 제출 정보 최종 리뷰 (오타, 깨진 링크)
