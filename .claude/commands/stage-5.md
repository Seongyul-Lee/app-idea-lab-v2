# Stage 5: 출시 + ASO (Launch & App Store Optimization)

대상 아이디어: $ARGUMENTS

MVP 빌드 완료 후, 앱스토어 최적화와 출시 전략을 실행한다. 스토어 페이지는 앱의 "영업사원"이다 — 여기서 전환율이 결정된다.

## 입력

- 스토어 제출 가능한 MVP 빌드
- `ideas/adopted/NNN-아이디어명-prd.md` — PRD 문서 (기능/타겟 유저 참조용)
- `ideas/adopted/NNN-아이디어명.md` — 평가 완료 아이디어 문서
- `validation/NNN-아이디어명-validation.md` — Stage 1V 검증 데이터 (있을 경우)

## 실행 절차

### Step 1: 키워드 리서치

스토어 검색 노출의 80%는 키워드가 결정한다.

**1) 핵심 키워드 추출**
- PRD의 pain point + 타겟 유저 조합으로 후보 키워드 20~30개 도출
- 예: 복약 관리 앱 → "약 알림", "복약 알림", "약 먹는 시간", "복약 관리", "투약 알림", "어르신 약", "매일 약", "처방전 관리" 등

**2) 키워드 분석**
- App Store: Apple Search Ads 기본 도구로 검색량/경쟁도 확인
- Play Store: Google Ads 키워드 플래너로 검색량 확인
- 서드파티 도구 (선택): AppTweak, Sensor Tower 무료 티어 활용

**3) 키워드 우선순위 매트릭스**

각 키워드를 아래 기준으로 채점:

| 기준 | 높음 (3점) | 보통 (2점) | 낮음 (1점) |
|------|-----------|-----------|-----------|
| 검색량 | 월 5,000+ | 1,000~5,000 | 1,000 미만 |
| 관련성 | 앱 핵심 기능과 직결 | 관련 있음 | 간접적 |
| 경쟁도 | Top 10에 소형 앱 다수 | 중형 앱 혼재 | 대형 앱 독점 |

→ 총점 7점 이상 키워드를 **Primary**, 5~6점을 **Secondary**로 분류

**4) 산출물**: 키워드 리스트 (Primary 5~8개, Secondary 10~15개)

### Step 2: 스토어 에셋 제작

`launch/store-assets-guide.md`를 참조하여 아래 에셋을 제작한다.

**필수 에셋 체크리스트**:

| 에셋 | App Store (iOS) | Play Store (Android) | 비고 |
|------|----------------|---------------------|------|
| 앱 이름 | 30자 | 30자 | Primary 키워드 포함 필수 |
| 부제목/짧은 설명 | 30자 (부제목) | 80자 (짧은 설명) | Secondary 키워드 포함 |
| 설명 | 4,000자 | 4,000자 | 처음 3줄이 가장 중요 |
| 키워드 필드 | 100자 (쉼표 구분) | 없음 (설명에 포함) | 앱 이름과 중복 금지 |
| 스크린샷 | 최소 3장, 권장 6장 | 최소 2장, 권장 8장 | 6.7" + 5.5" (iOS) |
| 앱 아이콘 | 1024×1024 | 512×512 | 단순, 원거리 식별 가능 |
| 카테고리 | Primary + Secondary | Primary | 경쟁 적은 카테고리 선택 |
| 프로모션 텍스트 | 170자 | — | 심사 없이 수시 변경 가능 |
| 프로모션 그래픽 | — | 1024×500 | 피처링 대비 필수 |

### Step 3: 앱 설명 작성

**구조 (첫 3줄이 생명)**:

```
[1줄] 핵심 가치 한 문장 — 유저가 얻는 결과 (키워드 포함)
[2줄] 구체적 사용 시나리오 또는 차별점
[3줄] 사회적 증거 또는 숫자 (있을 경우)
--- 여기까지가 "더 보기" 클릭 전 노출 ---

[핵심 기능 소개] — 기능 나열이 아닌, 유저 관점 혜택으로 작성
▸ (기능이 아니라) "매일 정해진 시간에 알림을 받아 약을 놓치지 않아요"
▸ (기능이 아니라) "가족의 복약 현황을 한 눈에 확인할 수 있어요"
▸ (기능이 아니라) "복약 기록이 자동 저장되어 병원 방문 시 보여드릴 수 있어요"

[프리미엄/구독 소개] — 있을 경우
[개인정보/고객지원 안내]
```

**작성 원칙**:
- 기능(Feature) 나열 금지 → 혜택(Benefit)으로 작성
- "~할 수 있습니다" → "~하세요" (행동 유도)
- Primary 키워드를 설명 내 3~5회 자연스럽게 포함
- 이모지는 구분자로만 사용, 과다 금지 (▸, ✓ 정도)

### Step 4: 스크린샷 기획

**스크린샷 스토리라인** (6장 구성 권장):

| 순서 | 내용 | 목적 |
|------|------|------|
| 1장 | **히어로 샷** — 앱의 핵심 가치 한 줄 + 대표 화면 | 첫인상. 3초 내 "이 앱이 뭔지" 전달 |
| 2장 | 핵심 기능 #1 — 가장 많이 쓰는 기능 | 주요 사용 시나리오 |
| 3장 | 핵심 기능 #2 — 차별화 기능 | 경쟁앱과 다른 점 |
| 4장 | 핵심 기능 #3 — 편의 기능 | 부가 가치 |
| 5장 | 위젯/알림/워치 등 — 플랫폼 기능 활용 | 생태계 통합 |
| 6장 | 사회적 증거 또는 CTA — "지금 시작하세요" | 전환 유도 |

**스크린샷 디자인 원칙**:
- 기기 목업 안에 실제 앱 화면 배치
- 상단 1/3에 캡션 텍스트 (큰 글씨, 핵심 혜택)
- 하단 2/3에 앱 화면
- 배경색은 앱 브랜드 컬러 (스토어에서 시선 끌기)
- 텍스트는 최대 10단어 이내

### Step 5: 출시 타이밍 전략

| 항목 | 권장 |
|------|------|
| 출시 요일 | 월~화요일 (주중 초반에 리뷰팀 활성) |
| 피해야 할 시기 | 앱스토어 피처링 시즌, 대형 앱 업데이트 직후 |
| 소프트 론치 | 먼저 Android에 출시하여 초기 피드백 수집 → iOS 출시 |
| 사전 주문 | iOS 사전 주문 활용 시 출시일 오가닉 부스트 가능 |

### Step 6: 출시 체크리스트 최종 점검

`launch/aso-checklist.md`의 전체 항목을 점검하고 미비 항목을 해결한다.

## 주의사항

- ASO는 한 번 하고 끝이 아니다 — 매월 키워드 리프레시 필수
- 스크린샷이 전환율의 60%를 결정한다. 개발보다 스크린샷에 시간을 더 쓸 것
- 첫 48시간의 다운로드 속도가 스토어 알고리즘 랭킹에 결정적 영향
- 별점 4.5 이상 유지가 장기적으로 가장 중요한 ASO 요소
