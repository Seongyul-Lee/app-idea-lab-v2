# Stage 6: 성장 전략 (Growth Strategy)

출시 후 초기 유저 확보와 리텐션 극대화를 위한 실행 전략을 수립하고 실행한다. **출시 당일부터 시작**하며, 출시 후 12주를 3개 페이즈로 나누어 운영한다.

## 입력

- `ideas/adopted/NNN-아이디어명-launch.md` (Stage 5 산출물)
- `validation/NNN-아이디어명-validation.md` (Stage 1V 검증 데이터, 있을 경우)
- 라이브 앱의 실제 지표 (설치수, DAU, 리텐션, 수익)

## 실행 절차

### Phase 1: 초기 부스트 (출시 후 1~2주)

**목표**: 첫 200~500 다운로드 확보, 리뷰 10개 이상 확보

**1) D-Day 론치 액션**

출시 당일 실행할 체크리스트:

- [ ] 개인 SNS (X, 인스타그램, 페이스북) 출시 알림 포스팅
- [ ] 관련 커뮤니티 3곳 이상에 출시 알림 (홍보가 아닌 "만들었습니다 + 피드백 부탁" 톤)
- [ ] Stage 1V에서 확보한 대기자에게 출시 알림 발송 (있을 경우)
- [ ] 지인 20명에게 개별 메시지 (설치 + 리뷰 요청)
- [ ] 개발자 커뮤니티 (GeekNews, 커리어리, OKKY) 출시 후기 글 작성

**2) 커뮤니티 시딩 (1~2주)**

`growth/community-marketing.md`를 참조하여 타겟 커뮤니티별 맞춤 전략 실행.

| 채널 | 전략 | 기대 효과 |
|------|------|----------|
| 에브리타임 | 해당 학교/학과 게시판에 "이런 앱 만들었는데 의견 부탁" | 대학생 타겟 시 고효율 |
| 블라인드 | 직장인 pain point 공감 글 → 댓글로 앱 자연 언급 | 직장인 타겟 시 |
| 네이버 카페 | 해당 주제 카페 가입 → 꾸준한 활동 후 앱 공유 | 니치 커뮤니티 |
| 클리앙 | IT/모바일 게시판에 개발 후기 형식으로 공유 | 얼리어답터 확보 |
| 레딧 (r/korea 등) | 한국 관련 서브레딧에 영문으로 공유 | 해외 한국인 유저 |
| X (트위터) | 개발 과정 스레드 → 출시 알림 | 개발자/얼리어답터 |

**핵심 원칙**:
- "홍보"가 아니라 "공유 + 피드백 요청" 톤 유지
- 커뮤니티 규칙 숙지 필수 (광고 금지 규정 확인)
- 한 커뮤니티에 1번만 포스팅 (스팸 방지)
- 댓글에 성실하게 답변 — 이것이 진짜 마케팅

**3) 리뷰 확보 전략**

앱 내 리뷰 요청 타이밍 설정:

```
트리거 조건 (AND):
1. 앱 사용 3일 이상 경과
2. 핵심 기능 2회 이상 사용 완료
3. 해당 세션에서 크래시 없음
4. 이전에 리뷰 요청 표시 안 함 (또는 30일 경과)

→ StoreKit (iOS) / In-App Review API (Android) 호출
```

- 절대 금지: 첫 실행 시 리뷰 요청, 기능 잠금 후 리뷰 요청
- 초기 10개 리뷰는 지인에게 정중히 요청 (별점 조작이 아닌, 솔직한 사용 후기)

### Phase 2: 성장 루프 구축 (출시 후 3~6주)

**목표**: DAU 성장 루프 1개 이상 확보, D7 리텐션 20% 이상

**1) 리텐션 분석 및 개선**

출시 2주 후 아래 지표를 측정하고 개선한다:

| 지표 | 목표 | 미달 시 액션 |
|------|------|------------|
| D1 리텐션 | 40%+ | 온보딩 플로우 개선, 핵심 가치 전달 속도 높이기 |
| D7 리텐션 | 20%+ | 푸시 알림 전략 (3일차, 5일차, 7일차), 습관 루프 설계 |
| D30 리텐션 | 10%+ | 주간/월간 리포트, 스트릭 기능, 커뮤니티 기능 |
| 세션당 시간 | 3분+ | 콘텐츠 깊이 추가, 발견(discovery) 요소 |
| 핵심 기능 도달률 | 60%+ | 온보딩에서 핵심 기능으로 바로 유도 |

**2) 오가닉 성장 루프 설계**

앱 내에 바이럴 루프를 최소 1개 심는다:

| 루프 유형 | 구현 예시 | 적합한 앱 유형 |
|----------|----------|--------------|
| **공유 루프** | 결과물 이미지/카드 SNS 공유 (앱 워터마크 포함) | 기록/트래커/테스트 |
| **초대 루프** | "친구 초대 시 프리미엄 1주 무료" | 소셜/협업 기능 있는 앱 |
| **콘텐츠 루프** | 사용자 생성 콘텐츠가 검색 엔진에 노출 | 커뮤니티/리뷰 앱 |
| **위젯 루프** | 홈 화면 위젯 → 앱 자연 노출 → 주변인 호기심 | 생산성/건강/날씨 |
| **알림 루프** | 유용한 알림 → 재방문 → 습관 형성 | 습관/리마인더/건강 |

각 앱에 맞는 루프 1~2개를 선택하여 MVP 이후 첫 번째 업데이트에 포함.

**3) 콘텐츠 마케팅 (저비용)**

| 채널 | 콘텐츠 유형 | 빈도 | 목적 |
|------|-----------|------|------|
| 블로그 (네이버/티스토리) | "[앱이 해결하는 문제] 해결법 5가지" SEO 글 | 주 1회 | 검색 유입 |
| X (트위터) | 개발/운영 비하인드, 업데이트 소식 | 주 3~5회 | 브랜드 인지도 |
| 인스타그램 | 앱 사용 팁, 비포/애프터, 유저 후기 카드 | 주 2~3회 | 비주얼 마케팅 |
| 유튜브 Shorts / 릴스 | 15~30초 앱 사용 시연 | 월 2~4회 | 바이럴 가능성 |

### Phase 3: 수익화 최적화 (출시 후 7~12주)

**목표**: 월 수익 목표 달성, 수익화 모델 검증

**1) 수익화 모델 점검**

| 모델 | 적합한 경우 | KPI |
|------|-----------|-----|
| **프리미엄 (1회 구매)** | 명확한 "잠금 해제" 가치가 있을 때 | 구매 전환율 3%+ |
| **구독** | 지속적 가치 제공 (데이터 동기화, 고급 기능) | 무료→구독 전환 2%+, 월 해지율 5% 이하 |
| **광고 (AdMob)** | 고빈도 사용 앱 (DAU 높음) | eCPM $3+, DAU 1,000+ |
| **광고 + 구독 하이브리드** | 대부분의 경우 권장 | 광고 제거 구독이 전체 수익의 30%+ |

**2) 페이월 최적화**

```
무료 유저 경험 설계:
├── 핵심 기능 100% 사용 가능 (가치를 먼저 느끼게)
├── 사용량/기간 제한 (예: 3개 항목까지 무료, 7일 무료 체험)
├── 프리미엄 기능 "맛보기" (1회 체험 후 잠금)
└── 자연스러운 업셀 시점:
    ├── 무료 한도 도달 시
    ├── 고급 기능 시도 시
    └── 7일 사용 후 (가치 인식 완료 시점)
```

**3) 가격 전략**

| 구독 주기 | 한국 시장 권장가 | 비고 |
|----------|-----------------|------|
| 주간 | ₩1,900~₩2,900 | 높은 LTV지만 해지도 높음 |
| 월간 | ₩3,900~₩6,900 | 가장 일반적 |
| 연간 | ₩29,900~₩49,900 | 월간 대비 40~50% 할인으로 유도 |
| 평생 | ₩59,900~₩99,900 | 초기 현금 확보용, 장기적으로는 구독보다 불리 |

## 주의사항

- Phase 1에서 리뷰/피드백이 부정적이면 Phase 2 전에 핵심 문제부터 해결할 것
- 마케팅 채널은 한 번에 1~2개에 집중 → 성과 측정 → 확장/폐기 판단
- "DAU가 늘지 않는 앱에 수익화 최적화를 하는 것"은 의미 없음 — 순서를 지킬 것
- 출시 후 첫 4주의 유저 피드백이 앱의 방향을 결정한다 — 가장 집중해서 들을 시기
