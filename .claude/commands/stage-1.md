## 1단계: 아이디어 발굴

$ARGUMENTS

## 목적
한국 시장의 실제 pain point를 기반으로 앱 아이디어를 도출한다.

## 사용 도구
| 도구 | 역할 |
|------|------|
| MCP: `naver-search` | 네이버 블로그/카페/뉴스에서 한국 사용자 불만·수요 수집 |
| MCP: `brave-search` | 글로벌 트렌드 및 해외 유사 앱 탐색 |

## 절차

### 1. 탐색 주제 설정

아래 25개 탐색 주제 중 사용자가 선택할 수 있도록 안내한다.
AskUserQuestion은 질문당 최대 4개 옵션을 지원하므로, **탭 방식**으로 주제를 탐색한다.

#### 탭 기반 선택 흐름

**첫 번째 탭** (AskUserQuestion):
아래 4개 옵션을 제시한다. 사용자는 위 전체 목록의 번호나 주제명을 "Other"로 직접 입력할 수도 있다.

| 옵션 | 설명 |
|------|------|
| 트렌드 기반 분석 | naver-search/brave-search로 최신 화제 키워드를 자동 탐색하여 아이디어 후보 도출 |
| 직접 입력 | 사용자가 원하는 주제를 자유롭게 입력 |
| 생활/소비 카테고리 | A~B 카테고리 (3~11번) 세부 주제 탐색 |
| 직장/성장/건강/취미 카테고리 | C~G 카테고리 (12~25번) 세부 주제 탐색 |

**두 번째 탭** (카테고리 선택 시):
선택된 카테고리 범위 내에서 AskUserQuestion으로 세부 주제 4개를 제시한다.
4개를 초과하는 주제는 질문 전에 텍스트로 전체 목록을 안내하여 "Other"로 선택 가능하게 한다.

#### 특수 분기

**"트렌드 기반 분석" 선택 시:**
1. `naver-search`의 `datalab_search` 및 `search_news`로 최근 1주일 인기 검색어·화제 키워드 수집
2. `brave-search`로 글로벌 앱 트렌드 및 Product Hunt 신규 앱 확인
3. 발견된 트렌드 중 앱 아이디어로 발전 가능한 후보 3개를 AskUserQuestion으로 제시
4. 사용자가 선택한 후보로 절차 2번 진행

**"직접 입력" 선택 시:**
AskUserQuestion의 "Other" 안내 또는 텍스트로 주제를 자유 입력받아 해당 주제로 절차 2번 진행.

### 2~6. 아이디어 도출 및 초안 작성

2. 선택된 주제에 대해 도메인별 pain point 탐색
3. `naver-search`로 네이버 블로그/카페/커뮤니티에서 실제 불만·수요 근거 수집
4. `brave-search`로 글로벌 트렌드 및 해외 유사 앱 존재 여부 확인
5. 제외 기준(법적 인허가, 기업 협약 필수, 1인 개발 불가) 사전 필터링
6. 아래 초안 작성 지침에 따라 아이디어 초안 작성

## 초안 작성 지침

### NNN 번호 부여
`ideas/` 디렉토리의 기존 파일을 확인하여 가장 높은 번호 + 1을 부여한다.

### 작성 범위
`ideas/_template.md` 기반으로 아래 섹션을 작성한다. 평가 점수·판정 섹션은 비워둔다.

| 섹션 | 작성 깊이 |
|------|----------|
| 한줄 요약 | 핵심 가치를 한 문장으로 |
| 해결하려는 문제 | 검색에서 수집한 실제 불만·수요를 근거로 기술 |
| 타겟 사용자 | 주요 타겟, 연령대, 규모 추정 |
| 핵심 기능 | MVP 범위 최대 3개 |
| 수익 모델 | 초기 구상 수준 (2단계에서 검증) |
| 경쟁 현황 | brave-search에서 확인한 유사 앱을 간략히 기록 (2단계에서 심화) |
| 기술 구현 검토 | RN+Supabase 적합성 간략 판단 (2단계에서 상세 평가) |

### 검색 근거 인용
수집한 검색 결과를 초안에 반영한다:
- **해결하려는 문제**: naver-search에서 발견한 실제 불만·수요 게시글을 근거로 인용
- **경쟁 현황**: brave-search에서 확인한 유사 앱·서비스명 기록
- 이 근거는 2단계에서 재수집 없이 활용된다
