# 성장 전략 플레이북

1인 개발 앱의 출시 후 12주 성장 전략. 각 Phase에서 사용할 구체적 전술과 판단 기준.

---

## Phase별 목표 요약

```
Phase 1 (1~2주)     Phase 2 (3~6주)     Phase 3 (7~12주)
초기 부스트      →   성장 루프 구축   →  수익화 최적화

목표:               목표:               목표:
· 500 다운로드       · DAU 성장 루프     · 월 수익 목표 달성
· 리뷰 10개+        · D7 리텐션 20%+    · ARPU 최적화
· 핵심 피드백 수집    · 오가닉 비율 50%+   · 구독 전환 2%+
```

## 온보딩 최적화

리텐션의 50%는 첫 실행 경험이 결정한다.

### 좋은 온보딩의 3원칙

1. **"아하 모먼트"까지 3단계 이내**: 유저가 핵심 가치를 체험하기까지 화면 3개 이하
2. **입력보다 선택**: 회원가입/설정에서 텍스트 입력 최소화. 선택지(버튼/칩) 제공
3. **스킵 가능**: 모든 온보딩 화면에 "건너뛰기" 옵션 제공

### 온보딩 플로우 템플릿

```
[화면 1] 가치 제안
  "이 앱으로 [결과]을 얻으세요"
  [시작하기] 버튼

[화면 2] 핵심 설정 (1가지만)
  앱의 핵심 기능에 필요한 최소 설정
  예: 알림 시간 설정, 카테고리 선택
  [다음] + [나중에 설정]

[화면 3] 핵심 기능 첫 사용
  유저가 직접 핵심 기능을 1번 사용하게 유도
  = "아하 모먼트" 달성
  
[메인 화면 도착]
  가이드 툴팁으로 추가 기능 안내 (비침투적)
```

### 측정 퍼널

```
앱 설치 → 첫 실행 → 온보딩 시작 → 온보딩 완료 → 핵심 기능 사용 → D1 재방문
  100%      85%         75%          55%          40%           35%

↑ 각 단계별 이탈률 측정 → 가장 이탈이 큰 단계를 최우선 개선
```

## 리텐션 전략 도구상자

리텐션 지표가 목표 미달일 때 적용할 수 있는 전술 목록.

### D1 리텐션이 낮을 때 (< 40%)

| 전술 | 구현 난이도 | 효과 |
|------|-----------|------|
| 온보딩에서 "아하 모먼트" 강제 체험 | 중 | ★★★★★ |
| 첫 실행 시 샘플 데이터 프리로드 (빈 화면 방지) | 하 | ★★★★☆ |
| 푸시 알림 허용 요청 타이밍 조정 (첫 실행 X, 가치 체험 후 O) | 하 | ★★★☆☆ |
| 첫 실행 후 12시간 뒤 로컬 알림 "어제 시작한 [작업] 이어가세요" | 하 | ★★★☆☆ |

### D7 리텐션이 낮을 때 (< 20%)

| 전술 | 구현 난이도 | 효과 |
|------|-----------|------|
| 드립 알림: 3일차, 5일차, 7일차에 유용한 팁 알림 | 중 | ★★★★☆ |
| 스트릭 시스템 (연속 사용 일수 표시) | 중 | ★★★★☆ |
| 주간 리포트 (알림 또는 이메일) "이번 주 당신의 기록" | 상 | ★★★★★ |
| 진행률 표시 "70% 완료! 조금만 더" | 하 | ★★★☆☆ |

### D30 리텐션이 낮을 때 (< 10%)

| 전술 | 구현 난이도 | 효과 |
|------|-----------|------|
| 월간 회고 리포트 "이번 달 당신의 변화" | 상 | ★★★★★ |
| 새 기능/콘텐츠 정기 업데이트 (2주마다) | 상 | ★★★★☆ |
| 커뮤니티/소셜 기능 (리더보드, 공유) | 상 | ★★★★☆ |
| 시즌/이벤트 콘텐츠 (월별 챌린지 등) | 중 | ★★★☆☆ |
| 이탈 유저 재활성화 알림 "돌아와주세요" (7일, 14일, 30일) | 중 | ★★★☆☆ |

## 바이럴 루프 설계

### 공유 가능한 콘텐츠 만들기

가장 효과적인 바이럴은 "유저가 자발적으로 공유하고 싶은 콘텐츠"를 만드는 것.

**공유 콘텐츠 체크리스트**:

- [ ] 유저의 성취/결과를 시각적으로 보여주는 카드 이미지
- [ ] 앱 이름 + 로고가 자연스럽게 포함된 워터마크
- [ ] 1탭으로 Instagram Story / X / 카카오톡 공유 가능
- [ ] 공유 이미지에 "나도 해보기" 링크 또는 QR 코드 포함
- [ ] 공유 시점: 목표 달성, 기록 완료, 결과 확인 등 "뿌듯한 순간"

**앱 유형별 공유 콘텐츠 예시**:

| 앱 유형 | 공유 콘텐츠 | 공유 트리거 |
|---------|-----------|-----------|
| 습관 트래커 | "30일 연속 달성!" 배지 카드 | 스트릭 달성 시 |
| 건강/운동 | 주간 운동 요약 카드 | 주간 리포트 생성 시 |
| 가계부 | 이번 달 절약 금액 카드 | 월말 정산 시 |
| 학습 | 학습 시간/완료율 카드 | 레벨 달성 시 |
| 기록/일기 | "올해 N일 기록 완료" 카드 | 특정 일수 달성 시 |

### 초대 시스템 (해당 시)

```
[유저 A] ──→ 공유 링크/코드 ──→ [유저 B]
    │                              │
    ▼                              ▼
  보상: 프리미엄 7일            보상: 프리미엄 3일
  (또는 특별 테마)              (또는 특별 테마)
```

- 양방향 보상이 단방향보다 2~3배 효과적
- 보상은 "시간 제한 프리미엄"이 가장 효율적 (한계 비용 0원)
- 초대 코드보다 딥링크가 사용자 경험 좋음

## 수익화 전환 퍼널

### 무료→유료 전환 최적화

```
무료 사용자
    │
    ▼ (핵심 가치 체험)
가치 인식 완료 (D3~D7)
    │
    ▼ (자연스러운 업셀 시점)
페이월 노출
    │
    ├── 무료 체험 시작 (7일)
    │       │
    │       ▼
    │   체험 종료 3일 전 알림
    │       │
    │       ├── 구독 전환 ✅
    │       └── 이탈
    │
    └── "나중에" 선택
            │
            ▼
        재노출 (14일 후, 새 기능 출시 시)
```

### 가격 실험

출시 초기에는 가격을 확정하지 말고, 2~4주 단위로 가격을 변경하며 전환율을 비교한다.

```
실험 1 (2주): 월 ₩3,900 / 연 ₩29,900
실험 2 (2주): 월 ₩4,900 / 연 ₩39,900
실험 3 (2주): 월 ₩6,900 / 연 ₩49,900

→ 전환율 × 가격 = 유저당 기대수익(ARPU) 최대 지점 선택
```

### 구독 해지 방어

구독 해지 시 아래 순서로 시도:

1. **해지 사유 설문** (1문항): "왜 해지하시나요?" → 데이터 수집
2. **할인 오퍼**: "50% 할인된 가격으로 계속 사용하시겠어요?" (iOS에서는 Offer Code 활용)
3. **다운그레이드 제안**: 연간→월간, 또는 더 저렴한 플랜 제안
4. **쿨다운 기간**: 해지 확정 후 7일 내 재구독 시 데이터 보존 안내

## 앱 업데이트 전략

### 업데이트 빈도

| 단계 | 빈도 | 내용 |
|------|------|------|
| 출시 후 1~4주 | 주 1~2회 | 크리티컬 버그 수정, 핫픽스 |
| 출시 후 1~3개월 | 2주마다 | 유저 피드백 기반 기능 추가, UX 개선 |
| 안정기 (3개월+) | 월 1회 | 새 기능, 시즌 콘텐츠, OS 대응 |

### 업데이트 릴리스 노트 활용

릴리스 노트는 **마케팅 채널**이다:

❌ 나쁜 예:
```
버그 수정 및 성능 개선
```

✅ 좋은 예:
```
🎉 v1.3 업데이트

여러분이 요청해주신 기능을 추가했어요!

✓ 다크 모드 지원 — 밤에도 눈이 편해요
✓ 위젯 크기 선택 — 홈 화면을 나답게
✓ 데이터 내보내기 — CSV로 한 번에

피드백 주신 모든 분들 감사합니다 🙏
더 나은 [앱 이름]을 위해 계속 달릴게요!
```
