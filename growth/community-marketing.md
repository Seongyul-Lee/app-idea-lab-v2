# 커뮤니티 마케팅 가이드

1인 개발자가 광고비 0원으로 초기 유저를 확보하기 위한 한국 커뮤니티 마케팅 실전 가이드.

---

## 핵심 원칙

1. **"홍보"하지 말고 "공유"하라** — 커뮤니티 유저들은 광고에 극도로 민감하다
2. **먼저 주고, 나중에 받아라** — 유용한 정보/콘텐츠를 먼저 제공한 뒤, 앱을 자연스럽게 언급
3. **1번만 포스팅하라** — 한 커뮤니티에 같은 앱을 2번 이상 올리면 스팸 취급
4. **댓글에 성실히 답하라** — 포스팅보다 댓글 대응이 10배 중요
5. **피드백을 진심으로 수용하라** — "그건 다음에 반영할게요" 식의 형식적 답변 금지

---

## 채널별 전략

### 1. 개발자 커뮤니티 (출시 후기 형식)

**적합한 앱**: 모든 앱 (개발자 커뮤니티는 "만든 과정" 자체에 관심)

| 채널 | URL | 특징 | 주의사항 |
|------|-----|------|---------|
| GeekNews | https://news.hada.io | IT/개발 뉴스. Show HN 스타일 공유 가능 | 기술적 내용 위주 |
| OKKY | https://okky.kr | 개발자 커뮤니티. 사이드 프로젝트 게시판 | 질 높은 글 기대 |
| 커리어리 | https://careerly.co.kr | 직장인/개발자 SNS | 개발 과정 스토리텔링 |
| 디스콰이엇 | https://disquiet.io | 메이커/스타트업 커뮤니티 | 프로덕트 등록 가능 |
| X (트위터) | — | 개발자/인디해커 커뮤니티 | 스레드 형식 효과적 |

**포스팅 템플릿: 개발 후기형**

```
[제목] 퇴근 후 1개월, 1인 개발로 [앱 이름] 만든 후기

1. 왜 만들었는가 (pain point + 동기)
2. 기술 스택 선택 과정 (Expo, Supabase 등 — 개발자들이 좋아하는 내용)
3. 만들면서 겪은 문제와 해결 (구체적 기술 이슈 1~2개)
4. 출시 후 첫 주 지표 (다운로드, 피드백 — 투명한 숫자 공유)
5. 다음 계획

앱 링크: (맨 마지막에 한 번만)
```

### 2. 타겟 유저 커뮤니티 (문제 공감 형식)

**적합한 앱**: 특정 유저 그룹의 pain point를 해결하는 앱

| 타겟 | 커뮤니티 | 접근 방식 |
|------|---------|----------|
| 대학생 | 에브리타임 | 학교/학과 게시판. "이런 거 불편하지 않아?" |
| 직장인 | 블라인드 | 직장 생활 불편 공감. 앱을 "해결책으로 발견" |
| 부모 | 맘카페 (네이버) | 육아 관련 고민 공감 → 도움 되는 앱 소개 |
| 건강 관심 | 네이버 카페 | 해당 질환/건강 관련 카페 활동 후 공유 |
| IT 얼리어답터 | 클리앙 | 앱/IT 게시판. 솔직한 리뷰 형식 |
| 일반 | 레딧 (r/hanguk 등) | 한국어 서브레딧. 캐주얼 공유 |

**포스팅 템플릿: 문제 공감형**

```
[제목] [pain point 공감 질문]?

[본문]
저도 이런 경험이 있어서 고민하다가,
직접 이 문제를 해결하는 앱을 만들어봤습니다.

(앱 핵심 기능 2~3줄 설명)

혹시 비슷한 고민 있으신 분들 의견 부탁드립니다.
사용해보시고 불편한 점 있으면 바로 반영할게요!

[앱 링크]
```

### 3. 콘텐츠 SEO (장기 유입)

**적합한 앱**: 검색 수요가 있는 주제의 앱

| 채널 | 콘텐츠 유형 | 목적 |
|------|-----------|------|
| 네이버 블로그 | "[주제] 해결법 5가지" | 네이버 검색 상위 노출 |
| 티스토리 | "[주제] 앱 비교 리뷰" | Google 검색 상위 노출 |
| 브런치 | 개발/기획 에세이 | 브랜드 스토리텔링 |

**SEO 블로그 글 템플릿**:

```
[제목] [키워드] 추천 앱 5가지 비교 (2025년)

1. [경쟁앱 A] — 장점/단점
2. [경쟁앱 B] — 장점/단점
3. [경쟁앱 C] — 장점/단점
4. [우리 앱] — 장점/단점 (객관적으로)
5. [경쟁앱 D] — 장점/단점

결론: 어떤 상황에 어떤 앱이 적합한지 추천

(자연스럽게 우리 앱의 차별점이 부각되도록 구성)
```

### 4. SNS (브랜드 인지도)

| 채널 | 콘텐츠 | 빈도 |
|------|--------|------|
| X | 개발 비하인드, 유저 피드백 공유, 업데이트 소식 | 주 3~5회 |
| 인스타그램 | 앱 사용 팁 카드, 비포/애프터, 유저 후기 | 주 2~3회 |
| 유튜브 Shorts | 15초 앱 사용 시연, "이런 기능 알고 계셨나요?" | 월 2~4회 |
| 스레드 | X와 동일 콘텐츠 교차 게시 | X와 동일 |

## 실행 타임라인

### 출시 당일 (D-Day)

| 시간 | 액션 |
|------|------|
| 오전 | 개인 SNS 출시 알림 (X, 인스타, 페북) |
| 오전 | 지인 20명에게 개별 메시지 (카톡/DM) |
| 오후 | 개발자 커뮤니티 출시 후기 포스팅 (GeekNews, OKKY, 디스콰이엇) |
| 오후 | Stage 1V 대기자에게 출시 알림 이메일 발송 |

### 출시 후 1주 (D+1 ~ D+7)

| 일차 | 액션 |
|------|------|
| D+1 | 타겟 커뮤니티 1곳에 문제 공감형 포스팅 |
| D+2 | 커뮤니티 댓글 전부 답변 + 피드백 수집 |
| D+3 | 타겟 커뮤니티 2곳에 포스팅 |
| D+4 | 급한 피드백 반영 핫픽스 (있을 경우) |
| D+5 | SEO 블로그 글 1개 발행 |
| D+6 | SNS 콘텐츠 (사용 팁 카드) 발행 |
| D+7 | 1주차 성과 정리: 다운로드, 리뷰, 피드백 요약 |

### 출시 후 2~4주

- 주 1회 SEO 블로그 글 발행
- 주 3회 SNS 콘텐츠
- 추가 커뮤니티 포스팅 (새로운 커뮤니티 1~2곳)
- 리뷰 답변 매일 체크

## 커뮤니티별 금기사항

| 커뮤니티 | 절대 하지 말 것 |
|---------|----------------|
| 에브리타임 | 외부 링크 직접 게시 (우회 방법 필요) |
| 블라인드 | 광고/홍보 의심 → 즉시 삭제 + 계정 제재 |
| 네이버 카페 | 가입 직후 바로 홍보 글 (활동 기록 없으면 삭제) |
| 클리앙 | 자작극 (자문자답) 적발 시 영구 차단 |
| 맘카페 | 남성이 여성 전용 카페 가입하여 홍보 |
| 디시인사이드 | 성의 없는 홍보 글 (커뮤니티 특성상 조롱당할 수 있음) |

## 효과 측정

각 채널의 효과를 측정하기 위해 UTM 파라미터 또는 채널별 고유 링크를 사용한다.

```
앱 랜딩 페이지 URL?utm_source=geekNews&utm_medium=community&utm_campaign=launch
앱 랜딩 페이지 URL?utm_source=blindApp&utm_medium=community&utm_campaign=launch
```

| 지표 | 측정 방법 |
|------|----------|
| 채널별 유입수 | UTM 기반 Analytics |
| 채널별 설치 수 | 시간대별 설치 스파이크로 추정 |
| 채널별 반응 | 조회수, 댓글수, 좋아요 수 |
| ROI (시간 투자 대비) | (해당 채널 설치수) / (투자 시간) |
